# Neo4j 構築・検証要件

## 目的
- Docker 上で Neo4j をローカル構築し、グラフ検索の実装と概念を理解する。
- 映画・ドラマ・アニメなどのコンテンツに対するレコメンドロジックを、実データに近い形で検証する。

## 想定シナリオ
1. 映像コンテンツのメタデータ（タイトル、ジャンル、出演者、制作年など）をノード・リレーションで表現する。
2. ユーザーの視聴履歴や評価をグラフ上に追加し、類似性・好みを判定するクエリを作成する。
3. コンテンツ間の類似度、俳優・ジャンル経由の推薦、時系列に基づくシリーズ提案など、複数のグラフクエリを実行する。

## 技術要件
- Docker Compose で Neo4j を起動できること（Bolt と HTTP ポートをホストへ公開し、ボリュームでデータ永続化）。
- ローカルから `cypher-shell` やアプリケーションコード経由でクエリを実行できること。
- サンプルデータ投入用のシードスクリプトまたは Cypher ファイルを用意し、環境初期化を繰り返し行えること。

## 検証要件
- 推薦クエリの結果をローカルで確認し、メトリクス（例: 類似度スコア、ヒット件数）を出力する。
- クエリパフォーマンスを確認し、必要であればインデックスや制約を設定する。
- 複数のクエリパターン（ユーザーベース、アイテムベース、ジャンルベースなど）を比較し、意図した推薦が得られるか評価する。

## 成果物
- Docker Compose 設定ファイルと起動手順。
- データモデル図または説明資料。
- サンプルデータ投入用の Cypher/スクリプト。
- 検証に使用したクエリと結果の記録（ログまたはレポート）。
